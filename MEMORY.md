# AgentFactory Memory

- Delivery watcher pattern: when a git-host PR is open but cannot be completed for a host-reported reason such as merge conflicts, do not leave the task in `delivering` and do not hide it under generic CI failure. Add a specific `failure/v1` reason, bounce to `queued`, and include same-branch repair instructions so the dispatcher reclaims the persisted branch and updates the existing PR.
- Dependency waiting is claim-readiness, not a universal task state: only Backlog/Queued dependents should display "Waiting on N". If a prerequisite is reopened after its dependent starts, the unmet count may rise but must not imply that active or completed work was interrupted.
- A successful Vite production build does not typecheck the React client. When a required `Task`/`TaskDetail` field is added, run `npm -w packages/web run typecheck:client` and update every typed fixture; preserve `exactOptionalPropertyTypes` by omitting optional props instead of passing explicit `undefined`.
- When a drawer navigates between entity keys without unmounting, guard async detail responses by both the current key and a request generation. Otherwise an older request can resolve last and pair the new header with stale body/actions.
- A supervisor Restart action must match the failure's lifecycle: dispatcher failures stay `queued`, reviewer failures stay `in_review`/`delivering`, and the owning supervisor must reconcile the `restart/v1` marker by clearing both its skip-list and attempt counter. Never swallow a restart API error in the drawer.
- On Windows, killing a spawned `.cmd` wrapper does not reliably kill its CLI descendant. Supervisor timeouts/shutdown must terminate the full process tree, clear reused final-output files before spawn, and accept only a completed Codex final-message artifact at the timeout boundary (never partial streaming stdout). Re-read that artifact in the timed-out reap path because it can appear while tree termination is in progress.
- SQLite migration slots are positional, not semantic. A deployed database can report the latest `user_version` after a divergent branch consumed that number while still lacking the expected schema; verify critical tables/columns, then reserve the next migration slot for an idempotent reconciliation that preserves unknown columns.
- Re-read a task's derived failure immediately before exercising restart behavior on a live supervisor. A fresh 1/N failure may supersede the earlier skip-listed N/N failure while a review is active; that task should auto-retry and correctly reject Restart, so use a currently `skipListed: true` task for the live proof.
- In PowerShell, quote Git revision syntax such as `'@{u}'`; unquoted `@{...}` is parsed as a hashtable literal before Git receives it.

Local-first agent orchestration · single operator

AgentFactory

A task board that is the state, memory, and control room for a fleet of AI coding agents. The board conducts — the agents build.

The board never runs an agent itself. It owns the rules, the lifecycle, and the review gates. Headless supervisor processes spawn fresh claude -p / codex sessions that claim work, write code in isolated git worktrees, and hand results back over MCP.

The core split

Two responsibilities, kept apart

Everything in the design flows from one boundary: the board is the source of truth and the authority; the agents are stateless workers that come and go. Neither reaches into the other's job.

The board owns

state · rules · review
  • ▸The lifecycle. A tiny transition table keyed by (from, to, actor) — the one place any new state edge is added.
  • ▸Persistent state. The only component that touches the database; tasks, activity, metrics, workspaces.
  • ▸Review & authority. Human-only gates: approve, delete, reopen, restart. Agents can never approve or delete.
  • ▸Attribution. Analytics and AI-review verdicts are derived from the log, so they work retroactively.

The agents do

claim · code · submit
  • ▸Claim one task. A session claims a queued task — the dispatcher spawns, the agent claims.
  • ▸Work in isolation. A per-task git worktree at .worktrees/<key> on branch feature/<key>-…
  • ▸Push, then hand off. The branch is pushed to origin before the result is submitted for review.
  • ▸Talk over a protocol. Board ops arrive as MCP tools — get_next_task, submit_result, report_progress.

The board is the persistent state store + UI for an external agent loop.

The engine room · three processes

Supervisors watch the board and act

The board is passive — it never spawns anything. Three headless processes poll it and drive the pipeline forward. They run as separate processes: one Ctrl-C stops all three, and a crash in one leaves the others running.

Why it holds together

Design principles worth stealing

A handful of decisions keep the system small, honest, and resilient to agents that crash, time out, or wander off.

Derive, don't store

The activity log is the truth

Metrics, AI-review state, and failures are derived from the log — never denormalized — so they work retroactively and can't drift.

Convention over column

Markers ride in comments

ai-review/v1, failure/v1, restart/v1 are parsed from comment bodies — one format, one place, no schema churn.

Stateless workers

One fresh session per task

No long-lived agent memory to corrupt. A crash just releases the claim; a retry starts clean from the pushed branch.

Isolation

A worktree per task

Each task gets its own .worktrees/<key> and branch, so parallel agents never collide in the working tree.

Cheap change signal

Poll a version string

getVersion() returns maxTimestamp#count. The UI and MCP diff it to refetch only when something actually moved.

Authority

Humans hold the gates

Agents advance and submit; they can never approve or delete. Delivery to done is verified by machine, not asserted by the worker.

A workspace binds tasks to a repo. Pinning the MCP server via AGENTFACTORY_WORKSPACE makes the agent a worker for exactly that repo — it can never claim another repo's tasks.

Six steps, click through

    You're running when…

    • ✓http://localhost:8787 shows the board, and the Workspaces button lists your workspace next to default.
    • ✓The MCP server logs connected over stdio (db: …, workspace: shopfloor) when the agent session starts.
    • ✓A queued task disappears from Queued and reappears under In Progress the moment the agent calls get_next_task — live, no refresh.
    • ✓The target repo gains .worktrees/AF-1/ on branch feature/AF-1-… (named by the server) while the agent works — and the worktree is gone again by the time the task reaches review.
    • ✓The task reaches In Review with a result summary and a branch link — pushed to origin and verified by submit guardrails. Your turn: Approve or Request changes.

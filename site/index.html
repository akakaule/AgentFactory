AgentFactory · human-supervised agent work, on a board

A local task board where humans create and review work in a web UI and agents claim and deliver it over MCP — Claude Code or any MCP client. The two sides never talk directly: one SQLite file is the only shared state. Every task is worked in an isolated git worktree on a server-named feature branch, pushed to origin before submission, and lands back on the board as a reviewable diff. Tasks can also arrive through producer agents that import real backlogs — Azure DevOps today via ado-bridge — and reflect every status change back to the source. The board is the human checkpoint; it never runs agents and never writes to a repo.

The loop, in one pass

  1. 1A task is created & queued — by a human on the board, or by a producer agent importing a backlog item and writing the spec (producers). Title, spec, acceptance criteria, workspace; only queued is claimable.
  2. 2Agent claims over MCP — get_next_task, FIFO within its pinned workspace. The payload carries the spec, prior feedback, pasted images, repoPath — and a protocol block: the server-named branch, the worktree path, and the exact finish steps. Claim-time data can't go stale.
  3. 3Isolated worktree — .worktrees/<key> on feature/<key>-<kebab-title>; parallel agents never collide. Progress is narrated with add_comment, live on the board via SSE.
  4. 4Finish protocol — commit → git push -u origin → remove the worktree. The pushed branch is the durable record.
  5. 5submit_result — guardrails verify the branch is on origin, fully pushed, and the worktree is gone — then the task lands In Review with summary, branch link, and token metrics. Unpushed work is refused with the commands to fix it.
  6. 6Human checkpoint — review the diff on the board. Approve → done; Request changes → re-queued with feedback, next claim continues on the same branch. PR opens from the branch; CI fails → comment + Reopen.

Guarantees & recent changes

What the system guarantees

◎One SQLite file is the only shared state — UI (HTTP), agents (MCP), and bridges all converge on it; SSE fans changes out live.
⚙Transitions are enforced in core — agents can never approve, reopen, or invent tasks; humans own every gate. Atomic claims: no double-claiming, ever.
⛓Convention is contract — the server names the branch at claim time and refuses submission until the work is pushed and the worktree cleaned. All checks are git reads: the board never writes to a repo.
↩Every dead end has a rescue loop — Request changes (in_review → queued), Reopen (done → queued), Release claim (in_progress → queued). History survives every round-trip.
⇄Scale by adding pairs — each repo = one workspace + one pinned worker. Queues never leak across workspaces.

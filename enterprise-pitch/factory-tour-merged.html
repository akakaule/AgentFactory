Every edge is declared

State changes are keyed by (from, to, by: human | agent). Nothing undeclared can happen.

Humans hold the gates

Approve, reopen, force-complete are human-only. Agents can never delete or approve — by construction.

Delivery is verified

A watcher confirms the PR is merged and CI is green — or the task goes back to the queue. No agent can fake "done."

The transition · L1 → L5

You move up the levels by tightening or loosening a gate — not by a rewrite

1

Assisted

Human writes; AI autocompletes.

2

Copilot

AI on a task under constant steering.

3

Bounded

Whole tasks in guardrails; human approves each.

4

Orchestrated

Many tasks in parallel; review by exception.

5

Self-delivering

Queue → verified delivery, minimal human touch.

Autonomy is a property of the deployment

The same agent is safe or reckless depending on the controls around it. AgentFactory is those controls — so you climb the levels without climbing the risk.

Where the gates sit defines your level

A pilot starts at L3 — review everything — and earns its way to L4/L5. That's a configuration decision, task by task, not a leap of faith.

For strict IT

Built to sit inside your controls

🔎 Auditable

Metrics, analytics and review state are all derived from an append-only activity log — a complete, replayable chain of custody per task. AI-review findings are walled off from the implementing agent.

🛡️ Self-hosted & resident

Runs on your infrastructure. State in a local database; git-host auth via env tokens — no secrets in the board. Each task runs in its own isolated git worktree.

🔗 CI/CD-native

PR-based flow by convention. Delivery verified on GitHub or Azure DevOps: merged + green, or it bounces back to the queue.

It's single-user and local by design today — and that's exactly what the co-build track extends, without loosening any of these guarantees.

Appendix — architecture

How it's built

One DB owner

Six packages, npm workspaces, strict TypeScript. A single package owns the database and every lifecycle rule — the state machine lives in one place.

Three supervisors

Dispatcher spawns one fresh agent session per queued task. Reviewer posts an advisory second-agent verdict. Watcher verifies delivery on the git host over REST — no LLM.

Local & isolated

Task state is a local database; each task runs in its own git worktree. Nothing about the work has to leave your network.

For deep IT or compliance questions, we offer a separate technical review of the ALP spec with your security team.
